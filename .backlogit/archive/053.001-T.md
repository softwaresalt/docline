External file-ref resolver + security containment. Extend the loader to resolve `file.json#/pointer` refs: load the target file relative to the referring file's directory, resolved through docline.paths.safe_workspace_path so `../`/absolute/symlink escapes above the workspace root are rejected (PathContainmentError). DENY URL-valued refs (http/https) — refuse, never fetch (SSRF). Apply 051-F fragment mapping (#/definitions/X -> #/components/schemas/X) to 2.0 targets. Cross-file cycle guard. Skip example (x-ms-examples) refs.

- A `./target.json#/definitions/X` ref resolves to the object in the sibling file.
- A `../common/target.json#/...` ref resolves within the workspace root.

- A `https://host/spec.json#/...` ref is denied (never fetched) with a typed error (test).
- A cross-file ref cycle terminates (guard, test).
- 2.0 external target fragments are mapped to components pointers.
