Add new function resolve_pdf_engine_for_file(path, *, requested) in src/docline/readers/pdf.py that wraps _resolve_layout_engine and additionally consults resource_probe.probe(). Behavior: when requested=='auto' AND probe.recommended_docling_max_pages > 0 AND file_size_mb <= probe.recommended_docling_max_mb AND (page_count is None OR page_count <= probe.recommended_docling_max_pages), returns 'docling'. Otherwise returns 'heuristic' with a reason string (file_too_large, page_count_too_high, insufficient_ram). For now, page count is approximated from file size (1 MB ≈ 7 pages heuristic) when pypdf isn't available; later wire to splitter for exact count. Existing _resolve_layout_engine semantics preserved for callers that don't pass a path. Tests: parameterized cases hitting each gate condition with mocked probe and tmp_path PDFs of various sizes. Closes stash 4B913619 implementation piece.
