Parse TOC.yml files (Microsoft DocFx convention) to derive a topological ingestion order over the corpus. TOC.yml is a list of entries each with {name, href, items[]} where href references a markdown file and items is an optional nested list. Walk the tree to produce a flat ordered list of href paths. Multiple TOC.yml files (one per major subdir, plus a root TOC.yml) merge in lexicographic subdir order. Emit the ordering as a crawl-manifest.json-equivalent so the existing _ordered_staged_files in app.py uses it. New helper src/docline/process/toc_parser.py with parse_toc(toc_yml_path, base_dir) → list[str] paths. Wire into a NEW staging-config option (e.g., FetchRequest.toc_yml_files: list[str]) — out of scope for this task is the actual fetch flow (which is 023-F further work).
