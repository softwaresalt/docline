<!-- BEGIN:description -->
Implement the 5-pass orchestrator `process_pdf_triaged()` in `src/docline/process/pdf_triage.py`, mirroring the shape of `process_pdf_in_chunks` from `pdf_batch.py`.

Five passes: (1) heuristic baseline across whole PDF via `read_pdf_pages(layout_engine='heuristic')`; (2) score each page via `fidelity_scorer.score_page`; (3) coalesce flagged indices via `page_range.coalesce_ranges`; (4) splice each range into a temp PDF using `pypdf.PdfWriter` and run the existing `docling_worker` subprocess on each splice; (5) merge per-page outputs into a final list where flagged pages come from docling and the rest from heuristic.

Returns a frozen `TriageResult` dataclass with `pages: tuple[str, ...]`, `engine_per_page: tuple[str, ...]`, `flagged_ranges: tuple[tuple[int, int], ...]`, `metadata: dict`.

Files:
- `src/docline/process/pdf_triage.py` (new, ~200 LOC)
- `tests/process/test_pdf_triage.py` (new)

Depends on: U1 (scorer), U2 (coalescer).
