Implement one-shot CLI: `docline ingest local-dir <path> --output <dir>` that functionally mirrors the YAML manifest flow.

Files: src/docline/cli.py + new dispatch in src/docline/app.py.

Implementation: generate a ManifestLocalSource(type=local, id=derived, path=path, include=include) in memory, wrap as SourcesManifest, call execute_elt_fetch against a temp staging dir, then execute_process against staging->output dir. Cleanup staging dir unless --keep-staging.

Parity requirement: same code paths as YAML manifest -> identical behavior. CLI is ergonomics, not a replacement surface.
