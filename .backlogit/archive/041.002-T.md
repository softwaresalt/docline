Add a new pure module src/docline/runtime/ocr_budget.py holding the calibrated coefficients and the cap/scale math. No docling/psutil imports — pure functions unit-tested with synthetic inputs (red-first).

<!-- BEGIN:acceptance-criteria -->
- Constants from docs/decisions/2026-06-30-ocr-memory-calibration.md: OCR_BASE_MB=1412.84, OCR_K_MB_PER_MPX=15.4942, OCR_PER_PAGE_FLOOR_MB=207.0, OCR_SAFE_FRACTION=0.6.
- predict_peak_mb(page_megapixels, scale, pages_per_group) implements base + k*mpx*scale^2*pages.
- max_ocr_pages_per_group(available_mb, *, page_megapixels, scale=2.0, safe_fraction=OCR_SAFE_FRACTION) returns min(bitmap-area cap, per-page-floor cap), never < 1.
- recover_scale_for_single_page(available_mb, *, page_megapixels, candidate_scales, safe_fraction) returns the highest scale whose single-page peak fits budget, or None.
- Tests (written first, observed failing): per-page-floor dominates at small mpx; bitmap cap dominates at large mpx*scale; scale recovery picks highest fitting scale; cap floored at 1 when even base overhead exceeds budget.
<!-- END:acceptance-criteria -->

Mirror the harness math in scripts/study/ocr_memory_calibration.py (recommend_max_pages / recommend_scale_schedule) but as typed production code with Google-style docstrings. budget = available_mb * safe_fraction; per-page-floor cap = (budget - base_mb) // per_page_mb. Keep the module import-light so resource_probe/page_range can import it without heavy deps.
