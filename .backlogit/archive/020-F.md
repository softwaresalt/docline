<!-- BEGIN:description -->
Close PA4 calibration from 021-S by shipping three interdependent fixes: (1) markitdown as the triage baseline engine (replaces pypdf for richer markdown), (2) Jaccard-similarity diff metric in QA tripwire (replaces the structurally-too-coarse _normalize_markdown), (3) new signal_layout_complexity in fidelity_scorer (detects mangled tables by inspecting source PDF layout structure rather than heuristic output text).

Source decision: docs/decisions/2026-06-07-pa4-calibration-closure.md
Plan: docs/plans/2026-06-07-pa4-calibration-closure-plan.md (Plan Review gate: ADVISORY)

Empirical evidence: PA3 cosmos run (2026-06-06) confirmed 11.4× wall-clock speedup at 3% flag rate; PA4 cosmos run (2026-06-07) surfaced 97% disagreement rate that manual inspection revealed was 4/5 measurement artifacts + 1/5 genuine scorer miss (page 470 RBAC table). markitdown bench (2026-06-07, logs/markitdown-bench/) confirmed markitdown produces real markdown lists/code/headings vs pypdf flat text but does NOT solve tables (its pdfminer.six backend has same limitation as pypdf — only docling's rt_detr ML model reconstructs the table).

Default `--pdf-mode auto` behavior unchanged. Triage-mode users get richer baseline + meaningful disagreement metric + table-aware scoring.
<!-- END:description -->
