Add `signal_layout_complexity(text: str, page_metadata: object | None = None) -> float` to fidelity_scorer.py. Inspect the source `pypdf.PageObject` for text-run X-coordinate clustering (column count via x-quantization with tolerance ~10 PDF units), line density vs page area, and grid-like positioning. Fires (returns positive value in [0, 1]) when X-cluster count is significantly higher than what the heuristic-extracted flat text would suggest — i.e., the source has columns/grid that pypdf flattened.

Returns 0.0 when page_metadata is None (charitable-when-no-metadata pattern from 021-S U1 signal_char_density).

Add 'layout_complexity' to _SIGNAL_NAMES tuple. Add 'layout_complexity': 1.1 to _DEFAULT_SIGNAL_WEIGHTS (slightly above char_density at 1.0 — layout structure detection is high-confidence when it fires).

<!-- BEGIN:implementation-notes -->
Plan-review P3#3 finding: narrow the exception handling. pypdf content-stream APIs vary across versions; catching specific exceptions makes real bugs visible while still being defensive.

Implementation approach for X-cluster detection:
1. `getattr(page_metadata, 'get_text_extraction_data', None)` or equivalent — pypdf doesn't expose text-run positions cleanly; may need to walk the content stream
2. Alternative: use `page.get_contents().get_data()` to get the raw PDF operators, parse Tm/Td/TD operators to extract X-positions of each text run
3. Cluster X-positions with tolerance 10 PDF units (1/7.2 inch); count distinct clusters
4. Compare cluster_count vs flat-text-line-count: signal = max(0, min(1, (cluster_count - line_count_estimate) / 4.0))

Constructing the multi-column fixture PDF: use `pypdf.PdfWriter` + raw content stream injection. Example:
```python
writer = pypdf.PdfWriter()
writer.add_blank_page(width=612, height=792)
page = writer.pages[0]
# Inject Tm + Tj operators for text at distinct X positions
```
This is the hardest part of the unit; budget extra time. If hand-crafting proves too brittle, capture as a follow-on stash and use a synthetic mock for the test instead.

Fallback: if walking content stream is too fragile, use pypdf's `extract_text(extraction_mode='layout')` output's whitespace patterns as a proxy — count distinct horizontal text bands per line.
