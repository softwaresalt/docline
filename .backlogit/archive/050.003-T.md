Render one Markdown document per operation. For each path + HTTP method, emit a document with H1 = `METHOD path` and H2 sections: Summary/Description, Parameters (as a table), Request body, Responses, Security. Per-operation granularity is the operator-confirmed v1 default (yields RAG-sized chunks and clean graph nodes). This task produces the operation Markdown body only; frontmatter/assembly is T5 and schema docs/edges are T4.

<!-- BEGIN:acceptance-criteria -->
- One Markdown document is produced per operation (path + method).
- H1 is `METHOD path` (e.g. `GET /reports/{reportId}`); H2 sections cover Summary/Description, Parameters, Request body, Responses, Security.
- Parameters render as a Markdown table (name, in, type, required, description).
- A golden-file test against a small hand-written OpenAPI 3.1 spec matches the rendered output byte-for-byte (deterministic ordering).
- Operations with no parameters / no request body render valid Markdown without empty-section noise.
<!-- END:acceptance-criteria -->
