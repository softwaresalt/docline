Coordinate ProgressReporter's in-place TTY progress line with interleaved logging so a log.warning emitted mid-progress does not corrupt the NORMAL-verbosity carriage-return line. Add ProgressReporter.clear() to erase the active CR line, a logging handler that clears before emitting each record, and a coordinate_logging() context manager wired into the CLI dispatch. TTY-only; non-TTY/piped output is unaffected. Follow-up from 052-S / PR #156 Copilot review; final polish before the v1 release.
