Add `--batch BATCH_MANIFEST_JSON` mode to `docline._tools.docling_worker`. Manifest schema: `{"chunks":[{"input":"path","output":"path"},...]}`. Worker imports docling and constructs DocConverter ONCE, then iterates the manifest writing one envelope per chunk to its output path. On per-chunk failure (docling raises mid-iteration), write `{"schema_version":1, "pages":[], "page_count":0, "text":"", "error":"<repr>"}` to that chunk's output path and continue. Process exit code 0 if at least one chunk succeeded; non-zero only if docling import or initial model load failed (so the parent decides whether to retry single-chunk or fall back to heuristic). Update `pdf_batch.process_pdf_in_chunks` and `pdf_triage._splice_and_run_docling` to use batched mode when N≥2 chunks; gate via opt-in kwarg (default True) for cautious rollout. Tests: (a) 3+ chunks back-to-back produce identical output to single-chunk path; (b) one-chunk-fails-others-succeed: failed chunk gets error envelope, others get pages; (c) docling-import-fails: process exits non-zero before writing any output; (d) opt-in flag disables batching and falls back to per-chunk loop; (e) consumers in pdf_batch/pdf_triage handle the error envelope shape. Plan reference: §T3. Empirical: record cosmos PA3+PA4 wall-clock before/after for the closure doc (target ≥30% reduction from cold-start dedup).
