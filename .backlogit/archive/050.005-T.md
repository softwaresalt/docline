Assemble rendered operation and schema bodies into complete, schema-valid `BaseDocument` outputs. Populate `BaseFrontmatter` with `doc_type="openapi_operation"` for operations and `doc_type="openapi_schema"` for component schemas; set `source` to the spec URI + `operationId` (schemas keyed by component name); compute `content_sha256`; carry the standard `chunk_strategy="h1-h2-h3"` and `schema_version`. This is the assembly seam that binds T3/T4 output to docline's output contract.

<!-- BEGIN:acceptance-criteria -->
- Every emitted operation doc has `doc_type="openapi_operation"`; every schema doc has `doc_type="openapi_schema"`.
- `source` = spec URI + `operationId` for operations (and spec URI + component name for schemas); `content_sha256` is computed over the body.
- Emitted documents pass `BaseDocument` / `BaseFrontmatter` schema validation (schema-validation test).
- `chunk_strategy` and `schema_version` match the existing contract defaults.
- Operations lacking an `operationId` fall back to a deterministic derived id (documented) rather than failing.
<!-- END:acceptance-criteria -->
