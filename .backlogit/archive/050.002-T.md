Build the spec loader and a local `$ref` resolver. Load the OpenAPI document from JSON or YAML (PyYAML — already a dependency) into a typed object model. Resolve local `#/components/*` (OpenAPI 3.x) references with a cycle guard so nested and circular refs do not cause infinite recursion. External/split-file `$ref` resolution is explicitly DEFERRED beyond v1 (it is a security boundary — file refs must stay within the workspace root and URL refs are an SSRF vector; do not introduce it here).

<!-- BEGIN:acceptance-criteria -->
- A spec loads from both `.json` and `.yaml` inputs into the same in-memory model.
- A nested local `$ref` (schema referencing another component schema) resolves to the target object.
- A circular local `$ref` (A -> B -> A) resolves without infinite loop and without stack overflow (cycle guard asserted by test).
- An unresolvable local `$ref` raises a typed, specific exception (not a bare except / KeyError leak).
- External/split-file `$ref` values are left unresolved and flagged/recorded (not fetched) — a test asserts they are not dereferenced.
<!-- END:acceptance-criteria -->

<!-- BEGIN:implementation-notes -->
- Depends on T1 (SourceKind.OPENAPI + routing seam).
- Only `#/components/*` local pointers are in scope for v1; `azure-rest-api-specs` splits across files (top follow-up risk) — do NOT resolve external refs.
- Cycle guard: track visited ref pointers on the resolution path; return the already-resolved/placeholder node on re-entry.
- Use a typed exception in docline's custom exception hierarchy for unresolved/invalid refs (Principle I — no bare except).
- Reference: spike design table '$ref resolution' row + Risks 'External-$ref resolution is a security boundary (deferred)'.
