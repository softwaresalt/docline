<!-- BEGIN:implementation-notes -->
Resolve tag→SHA via `gh api repos/actions/upload-artifact/git/ref/tags/v7.0.1` and likewise for download-artifact v8.0.1. Downloads use `name: dist` (by name), so the v5 by-ID path change does not apply. **`download-artifact@v8` changes digest-mismatch from a warning to a workflow failure by default** (configurable via the new `digest-mismatch` input) — this applies to our `dist` download too. Because upload and both downloads run in the same workflow the digest always matches, so no spurious failure is expected; accept the secure `error` default (do NOT set `digest-mismatch: warn`). v8 is also ESM (transparent) and skips decompression for non-zip downloads (our artifact is a zip, so unaffected). Cannot be exercised on a normal PR (tag-triggered workflow); final proof is a clean next-tag Release run with no Node 20 annotations.
