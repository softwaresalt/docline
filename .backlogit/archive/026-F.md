Three independent quality improvements to the source-MD ingest pathway, derived from gaps discovered while ingesting the 1,340-file Power BI corpus (docs/decisions/2026-06-09-powerbi-corpus-coverage.md) and 027-S empirical findings (docs/closure/027-S-local-dir-ingest.md "T3 empirical finding"):

T1 — Auto-apply heading-disorder tolerance for include-fragment files (consumes 013FAC8A). Auto-detect files where the body has no H1 anywhere (the Microsoft Learn include-fragment convention) and selectively bypass the heading-hierarchy validation for those files only. Preserves the strict-mode quality-signal feedback loop for normal content. Eliminates the 16 remaining Power BI strict-mode frontmatter failures without requiring the operator to pass --allow-heading-disorder corpus-wide.

T2 — DocFx tabbed content handler (consumes 2C74D31B). Recognize ### [Label](#tab/key) pattern used by Microsoft Learn for tab UI. Either flatten to linear sections preserving label as section anchor OR emit as nested code-fenced tab block. Affects developer/visuals/dynamic-drill-down.md, developer/visuals/highlight.md and similar files.

T3 — Cross-product absolute-path link extraction (consumes 8A5D3AC2). Extend cross_doc_links regex to capture [text](/path/...) forms (without .md extension), emit with cross_product: true marker in the typed edge dict. Power BI corpus contains ~416 inline /fabric/ refs plus hundreds more for /dax/, /azure/, /power-platform/ that are currently invisible to the graph layer.

All three ship together as one PR + one CI run (CI is currently paused; future re-enable will use ubuntu-only matrix per ci.yml comment block).

Verification gated by:
- AC1: T1 lifts Power BI strict-mode rate from 98.8% to 100% (eliminates 16 known-good include-fragment failures)
- AC2: T2 emits valid markdown for the 4 known tabbed-content files without losing semantic structure
- AC3: T3 extracts cross_product=true edges for /fabric/, /dax/, /azure/, /power-platform/ paths in the Power BI corpus (expected ~1,000+ new typed edges)
- AC4: All four local quality gates pass (ruff check, ruff format --check, pyright src/, pytest)
- AC5: Existing 1113-pass test suite + new TDD tests pass; no regressions
