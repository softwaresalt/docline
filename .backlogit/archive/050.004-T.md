Render one Markdown document per named component schema, and emit `operation -> schema` cross-references as Markdown links so graphtor harvests typed referential edges. Each `$ref` from an operation to a component schema becomes a Markdown link whose target resolves to the corresponding schema document. This reuses docline's existing referentiality harvesting (no new contract surface).

<!-- BEGIN:acceptance-criteria -->
- One Markdown document is produced per named component schema.
- Each `operation -> schema` `$ref` is emitted as a Markdown link in the operation document.
- A test asserts every emitted link target resolves to an actual schema document (no dangling links).
- A test asserts the graphtor edge shape (source operation node -> target schema node) is what referentiality harvesting expects.
- Nested schema refs (schema -> schema) also emit links between schema docs.
<!-- END:acceptance-criteria -->
