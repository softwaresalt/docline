Add an opt-in page-aware stitch path in src/docline/process/pdf_batch.py that prepends `<!-- page N -->` markers before each page's markdown, so downstream graph writers (graphtor-docs) can recover page boundaries from the stitched output. Drive it from the per-page data already in ChunkResult.chunk_pages. Default off so current stitched output is unchanged. Source: 032-S T2 follow-up (stash FADDE6D5).

<!-- BEGIN:acceptance-criteria -->
- With markers disabled (default), stitched output is byte-identical to current behavior (existing tests unchanged).
- With markers enabled, each page's markdown is preceded by a `<!-- page N -->` comment; N is a monotonic 1-based counter.
- `page_overlap`-duplicated boundary pages are not double-counted/double-marked (or the deferral is documented with a deterministic counter).
- Heuristic chunks / empty or single-element `chunk_pages` degrade gracefully (documented — e.g. one marker per chunk).
- Existing H1 boundary-dedup behavior is preserved.
- Unit tests in tests/process/test_pdf_batch.py cover: disabled parity, multi-page-per-chunk numbering, two-chunk overlap dedup, and heuristic fallback.
- pytest / ruff / pyright green.
<!-- END:acceptance-criteria -->

- `_stitch_chunk_markdown(chunk_bodies)` @ src/docline/process/pdf_batch.py:449; called @ :209 as `_stitch_chunk_markdown([cr.markdown for cr in chunk_results])`. Page markers need per-page data, so thread `chunk_results` (with `chunk_pages`) into a page-aware stitch instead of only the joined `markdown`.
- Per-page markdown: `ChunkResult.chunk_pages` (tuple) @ :78, populated by `_parse_worker_envelope` @ :367 from the docling worker envelope `pages`. Empty for heuristic chunks; single-element `(raw,)` on the defensive non-envelope fallback.
- `split_pdf(path, max_pages=..., cache_dir=...)` @ :179 uses split_pdf's DEFAULT `page_overlap` (no arg passed). Overlap makes the first N pages of chunk K identical to the last N of chunk K-1 — skip those leading overlap pages when numbering to keep N source-relative. Obtain the overlap count from `docline.readers.pdf_splitter` (default constant) or thread split metadata.
- Recommended: add opt-in `page_markers: bool = False` on `process_pdf_in_chunks` (and the batch entry points), threaded into the stitch path; default off preserves current `"\n\n".join` output.
- Reuse existing boundary logic (`_drop_leading_h1_if_matches` @ :479, `_last_h1_title`) so H1 dedup still applies.
- If exact source-page numbering across overlap proves complex, an initial version may number post-dedup emitted pages monotonically and defer precise source-page mapping — flag it in the closure.
