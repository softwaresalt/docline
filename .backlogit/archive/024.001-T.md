Parse common DocFx container syntax into standard markdown so downstream consumers (graph extraction, embeddings, LLM context) see alt-text and structural intent rather than opaque syntax. Containers to support: :::image type='content' source='X' alt-text='Y'::: → standard ![Y](X) markdown image; :::row::: :::column::: → preserve as-is in body (DocFx layout is hard to translate losslessly); :::moniker range='X'::: ... :::moniker-end::: → strip the moniker wrappers but preserve content (the conditional logic was a publish-time concern; consumed source already shows the desired moniker variant); :::code language='X' source='./path.cs'::: → preserve as ```X code fence with the file reference in a comment. New helper module src/docline/process/docfx_normalize.py with a single normalize_docfx_containers(body, source_dir) function. Tests cover image, moniker, row/column, and unrecognized container pass-through.
