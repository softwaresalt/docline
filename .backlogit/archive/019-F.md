<!-- BEGIN:description -->
Introduce an opt-in `--pdf-mode triage` pipeline that runs the heuristic engine across the whole PDF for a fast baseline, scores each page with deterministic fidelity signals (char density, non-ASCII ratio, long-line/no-whitespace anomalies, multi-column gaps, table-char density, pypdf image/form-field metadata), coalesces flagged pages into ranges with a context buffer, splices just those ranges into temp PDFs via `pypdf.PdfWriter`, runs the existing `docling_worker` subprocess on each splice, and merges per-page outputs into a final list where flagged pages come from docling and the rest from heuristic.

Motivating evidence: `azure-cosmos-db.pdf` (3,426 pages, 47 chunks) takes ~9.5 h under the current all-docling path on the 2026-06-04 RCA host class. Triage mode is estimated at ~70–90 min (6–8× speedup) at equivalent fidelity on pages that need it.

Default behavior unchanged — `--pdf-mode auto` continues to invoke `process_pdf_in_chunks` exactly as today. Triage is opt-in only.

See plan: docs/plans/2026-06-06-triage-then-repair-plan.md (Plan Review gate: ADVISORY).
<!-- END:description -->

<!-- BEGIN:dod -->
All of the following must be true:

- New CLI flag `--pdf-mode triage` is available, documented, and orthogonal to existing `--pdf-engine` selection.
- Triage mode produces page-identical output to all-docling on flagged pages and to all-heuristic on unflagged pages.
- Per-page engine attribution is recorded in part frontmatter `docline:` namespace (merged, not overwritten) and as a manifest-level `triage_stats` summary.
- `--triage-report-only` mode emits a per-page TSV `(page_index, signals..., aggregate, needs_docling, reason)` without invoking docling.
- `--sample-rate FLOAT` QA tripwire mode re-runs that fraction of unflagged pages through docling, diffs against heuristic output, and records disagreement count in run metadata.
- Cosmos PDF wall-clock under triage mode ≤ 25% of the all-docling baseline.
- All existing pipeline modes (`auto`, `heuristic`, `docling`) work unchanged — verified by regression tests.
- All RED tests for each implementation unit pass GREEN.
- All four `ProposedAction` entries from plan hardening (PA1–PA4) reach `applied` or `abandoned`.
- Calibration TSV produced from running `--triage-report-only` against the 12 completed cosmos chunks; weights file locked in `src/docline/process/fidelity_weights.json`.
- Quality gates pass: `ruff check .`, `pyright src/`, `pytest`, `ruff format --check .`.
<!-- END:dod -->

<!-- BEGIN:goals -->
1. Cut wall-clock for long technical reference PDFs by ~6–8× without changing fidelity on layout-dense pages.
2. Record per-page engine attribution in the output contract so downstream RAG/graph consumers have a confidence signal.
3. Provide `--report-only` and `--sample-rate` validation modes so scorer thresholds can be calibrated empirically before production use.
4. Preserve bit-identical default-mode behavior; opt-in surface only.
