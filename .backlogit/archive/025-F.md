Enable a one-shot CLI ingest of local cloned docs repos that functionally mirrors the existing manifest/YAML config flow.

Surface: `docline ingest local-dir <path> --output <dir>` produces the same staging+processing result as a `type: local` ManifestLocalSource entry, but without requiring the operator to write a YAML config first. Both surfaces stay supported; this adds CLI ergonomics, not a replacement.

Decomposed into 5 tasks (T1-T5) per the plan at docs/plans/2026-06-10-local-dir-ingest-plan.md. Verification gated by 6 acceptance criteria (AC1-AC6); ships only when all six pass, including AC5 (Power BI corpus frontmatter rate >=99.5%, vs 98.8% baseline).

Consumes stash 0FC71B03 (frontmatter robustness fix). Defers 2C74D31B, 8A5D3AC2, 4A650FFD, F8E142A1, F10EB5CB to follow-on shipments.
