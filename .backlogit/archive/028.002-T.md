Add `PreTriageDecision` frozen dataclass (page_index, signals, aggregate, classification, reason) and `pre_triage_score(page_index, page_metadata, weights_path=None) -> PreTriageDecision` function to fidelity_scorer.py.

classification is Literal["route_to_docling", "route_to_heuristic", "uncertain"] with thresholds:
- aggregate >= 0.7 → route_to_docling
- aggregate <= 0.2 → route_to_heuristic
- else → uncertain

Signals: 5 source-aware (image_heavy, form_fields, layout_complexity, font_diversity, text_flow_consistency). Add _PRE_TRIAGE_SIGNAL_NAMES tuple and _DEFAULT_PRE_TRIAGE_WEIGHTS dict separate from the existing _SIGNAL_NAMES / _DEFAULT_SIGNAL_WEIGHTS. Use the existing load_weights helper with a new optional weights_path parameter to allow operators to override pre-triage weights independently of post-extraction weights.

TDD. Tests in tests/process/test_fidelity_scorer.py — 6 tests minimum:
- pre_triage_score returns route_to_docling on a page with multi-column + 3 fonts + 5 images
- pre_triage_score returns route_to_heuristic on a clean prose page (all signals near 0)
- pre_triage_score returns uncertain for an in-between page
- pre_triage_score respects weights_path override
- PreTriageDecision is frozen (mutation raises FrozenInstanceError)
- pre_triage_score is deterministic across two invocations on same input
