- `crawl()` accepts optional keyword `progress: Callable[[int, int | None, str], None] | None = None`.
- Callback fires immediately after each `page_count += 1` — on the 4 budget-consuming branches ONLY: robots-denied (crawl.py:158), fetch failure (177), domain-rejected redirect (190), and each successfully-emitted page (221); counts monotonic non-decreasing and ≤ max_pages.

- `max_pages` is a budget/ceiling: the crawl routinely ends early with `done < total` and forges no synthetic 100%.
- `progress=None` leaves the returned list of CrawlResult unchanged (characterization test).
- Tests: callback fires for a robots-denied/failed page (budget-consumed, not staged); a non-consuming URL (duplicate final / print page) fires NO callback and leaves page_count/budget unchanged; early frontier exhaustion → last event `done < total`, no forced 100%.
