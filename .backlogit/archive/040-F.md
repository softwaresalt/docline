Complete the OCR-OOM recovery arc from 038-F with HOST-RELATIVE, empirically-calibrated limits — no hardcoded page counts. 038.003-T recovers the accumulation case via group downsizing; a single page whose bitmap alone exceeds OCR memory still crashes.

OPERATOR DESIGN (2026-06-30, two clarifications):
1. The 8->4->2->1 pages and 0.5/0.25 scale steps were ILLUSTRATIVE, not literal. The shipped OCR_MAX_BATCHED_PAGES=8 and cap//2 halving are PROVISIONAL placeholders.
2. OOM risk is RELATIVE TO THE HOST's available memory. So calibration must NOT produce a fixed page cap for one machine. Instead it produces a PORTABLE memory-cost model — docling OCR peak memory as a function of page bitmap size (megapixels) x render scale, which is largely hardware-independent for the same computation. docline then runs an algorithm on ANY host: probe available memory (resource_probe), and compute the OCR per-group page cap (and, for an oversized single page, the render scale) that keeps peak memory within a safe fraction of THIS host's available memory. Same binary adapts: a 128 GB box caps higher than an 8 GB box.

This feature delivers the MEASUREMENT + the MECHANISM PLUMBING (agent-buildable, unit-testable): (1) an ocr_scale knob through the docling worker; (2) a calibration harness (operator-run on a docling host) that measures OCR peak RSS across page-size x scale and emits a portable cost model + coefficients, recording the measurement host's available memory for normalization context. APPLYING the model — a runtime host-relative cap/scale algorithm that replaces the provisional constant — is the follow-up (stash 699FB5DC) after the operator runs the harness, mirroring the 036-S harness -> operator run -> apply pattern.

Grounded surface: src/docline/readers/pdf.py::_read_pdf_docling_pages (PdfPipelineOptions images_scale); src/docline/_tools/docling_worker.py (--batch manifest fields); src/docline/process/batch_dispatch.py (retry loop); src/docline/runtime/resource_probe.py (available_ram_gb, recommended_docling_max_mb — the per-host memory budget the cap is computed against at runtime).
