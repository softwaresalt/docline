Per stash EFC6C84E (high) and 2026-06-08 extraction study, invert the triage scoring model so the scorer fires BEFORE running heuristic extraction. Extract a pre-triage scorer using 5 source-aware signals (3 existing + 2 new: font_diversity, text_flow_consistency) that classifies each page as route_to_docling | route_to_heuristic | uncertain. Uncertain pages continue through the existing heuristic-then-score path. Opt-in via new --triage-pre-score flag for backward compatibility.

Acceptance criteria: AC1-AC7 per docs/plans/2026-06-11-invert-triage-scoring-plan.md. Key gate: AC5 enforces ≥85% agreement with docling's empirical wins from the 2026-06-08 study before rollout.

Decomposed into 4 tasks (T1-T4) totaling ~6.5h human-equivalent effort. Single thematic concern (pre-extraction scoring).
