Fix two 032-S regressions surfaced by the post-merge cosmos validation runs. (1) OBSERVABILITY GAP (Constitution V): docling-worker subprocess stderr was discarded at every call site in pdf_triage (splice batched + per-range paths, QA tripwire) and pdf_batch (batched + per-chunk), so the operator's cosmos run showed subprocess_fallback_count=86/86 (100% docling failure) with NO visible cause. Now logs the captured worker stderr / error-envelope detail at WARNING on every failure path. (2) QA ENVELOPE-PARSING MISS: the QA tripwire read the worker output with qa_md.read_text() as flat markdown, but post-032-S T1 the worker writes a JSON envelope — so similarity was computed against the raw JSON string, garbaging the QA disagreement metric whenever docling succeeds. Now parses the envelope text via new _worker_output_text() helper. Both fixes are TDD-tested with mock runners (no docling inference). Does NOT change the root cause of the operator's 100% docling failure (a docling convert-time error inside the worker subprocess in their environment) — the observability fix makes that cause visible on the next run. Plan: focused fix, no deliberation. Verdict: PROMOTE.
