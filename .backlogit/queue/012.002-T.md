Create `src/docline/process/segment.py` implementing `segment_markdown(markdown, *, max_chars=12_000) -> list[str]` per the plan's algorithm: H1 hard split, H2 sub-split when an H1 part exceeds `max_chars`, char-bin fallback when no H1 exists or when H2 sub-parts still exceed the limit. Use `markdown-it-py` token `map` slicing on the original markdown text to preserve fenced code, tables, and source whitespace.

- Strip leading/trailing whitespace per emitted segment; drop empty segments; return `[""]` if the final list is empty
