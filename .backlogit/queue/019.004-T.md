<!-- BEGIN:description -->
Add `--pdf-mode {auto,triage}` CLI flag to `src/docline/cli.py`. Default is `auto` (current behavior unchanged). When set to `triage`, the process phase routes PDFs through `process_pdf_triaged` instead of `process_pdf_in_chunks`.

The new flag is orthogonal to existing `--pdf-engine` selection: `--pdf-mode` is the orchestration strategy; `--pdf-engine` is the within-mode extractor choice.

Files:
- `src/docline/cli.py` (modified)
- `tests/cli/test_pdf_mode_flag.py` (new)

Depends on: U3 (orchestrator).
<!-- END:description -->

<!-- BEGIN:implementation-notes -->
Add the flag at the same argparse level as `--pdf-engine`. Document in CLI help that the two flags compose meaningfully — future modes may expose `--pdf-mode triage --pdf-engine docling` to re-run repair pass with engine variations.

The routing decision happens in the existing process-phase dispatcher; introduce a small router function rather than scattering `if mode == 'triage'` branches.
