Replace `_normalize_markdown` in pdf_triage.py with a Jaccard-similarity helper. New private function `_content_similarity(a: str, b: str) -> float` returns Jaccard similarity of lowercased + Unicode-aware-tokenized + punctuation-stripped token sets in [0.0, 1.0]. Empty inputs both empty → 1.0; one empty + one non-empty → 0.0.

Implement U2 BEFORE U1 in build-feature order per plan-review P3#5 sequencing finding.

Tokenization spec (plan-review P3#2): `re.findall(r'\w+', text.lower())`. Returns empty list on empty input. Unicode-aware. Handles whitespace + punctuation in one pass.

Jaccard formula: `|A ∩ B| / |A ∪ B|`. Edge cases:
- Both empty: union is empty → define as 1.0 (vacuously identical)
- One empty + one not: union is non-empty, intersection is empty → 0.0
- Use `set()` for both — duplicates don't affect Jaccard

Default threshold 0.7 chosen based on PA4 inspection: page 107 (code-fence formatting, same content) showed >0.9 token overlap; page 470 (heuristic broken text vs docling table) would show <0.3.

Histogram buckets fixed at {>=0.9, 0.7-0.9, 0.5-0.7, <0.5} for calibration consistency across runs.
