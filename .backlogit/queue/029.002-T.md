Add Mistral OCR support via raw httpx (Foundry MaaS endpoints are path-routed; mistralai SDK is incompatible with them).

**Discovery probe result (2026-06-13)**: tested mistralai>=1 SDK with `server_url=<full-foundry-endpoint>` against operator's Foundry deployment — fails with 404 because the SDK appends `/v1/ocr` to the already-routed endpoint. Raw httpx POST with `Authorization: Bearer <key>` and JSON body succeeds. Returns `{pages: [{markdown}], usage_info, content_filter_results}` structure.

Architecture decision: use raw httpx (lightweight, well-maintained, already a transitive dep) instead of the mistralai SDK. Cleaner — works against both Foundry path-routed endpoints AND any other endpoint that accepts the standard Mistral OCR REST body.

1. pyproject.toml: add `mistral = ["httpx>=0.27,<1"]` to [project.optional-dependencies]. NOTE: not mistralai — see discovery probe result above.
2. src/docline/dependencies.py: add `mistral_available() -> bool` helper (checks for httpx via importlib.util.find_spec)
3. New src/docline/readers/mistral.py:
   - Public API: `read_pdf_mistral(path: Path, api_key: str | None = None, endpoint: str | None = None, model: str = "mistral-document-ai-2505") -> str`
   - `MistralCredentialError(DoclineError)` raised when no credential pair is resolvable
   - Lazy httpx import via `require_extra("httpx", extra="mistral")` raising DependencyUnavailableError when [mistral] not installed
   - **Credential resolution order** (per AC4):
     a. Explicit `api_key` + `endpoint` args win
     b. Else AZURE_AI_FOUNDRY_KEY + AZURE_AI_FOUNDRY_ENDPOINT env vars (Foundry path — preferred)
     c. Else MISTRAL_API_KEY + (MISTRAL_OCR_ENDPOINT or default https://api.mistral.ai/v1/ocr) (direct Mistral fallback)
     d. Else raise MistralCredentialError
   - Optional MISTRAL_OCR_MODEL env var overrides the model arg default
   - Reads PDF bytes, base64-encodes, builds inline `data:application/pdf;base64,...` document_url
   - POSTs JSON `{"model": resolved_model, "document": {"type": "document_url", "document_url": data_url}}` with `Authorization: Bearer <key>` header
   - Parses response: concatenates `pages[].markdown` with double-newline separator
   - Per-call telemetry: log page count + wall time + response usage_info (Foundry may include cost details)
   - Defensive error handling: catch httpx.HTTPError / httpx.HTTPStatusError, re-wrap as DoclineError with path + status code context
   - Validate endpoint shape when set: must look like a URL (starts with https://). MistralCredentialError with clear message when malformed.
