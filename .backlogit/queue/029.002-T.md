Add Mistral OCR support — optional [mistral] extra + reader module with Foundry-first credential resolution.

1. pyproject.toml: add `mistral = ["mistralai>=1,<2"]` to [project.optional-dependencies]
2. src/docline/dependencies.py: add `mistral_available() -> bool` helper mirroring the (now-removed) `adi_available` shape (uses importlib.util.find_spec for `mistralai`)
3. New src/docline/readers/mistral.py:
   - Public API: `read_pdf_mistral(path: Path, api_key: str | None = None, server_url: str | None = None, model: str = "mistral-document-ai-2505") -> str`
   - `MistralCredentialError(DoclineError)` raised when no credential pair is resolvable
   - Lazy SDK import via `require_extra("mistralai", extra="mistral")` raising DependencyUnavailableError when [mistral] not installed
   - **Credential resolution order** (per AC4):
     a. Explicit `api_key` + `server_url` args win
     b. Else AZURE_AI_FOUNDRY_KEY + AZURE_AI_FOUNDRY_ENDPOINT env vars (Foundry path — preferred)
     c. Else MISTRAL_API_KEY env var with SDK default server_url (direct Mistral fallback)
     d. Else raise MistralCredentialError
   - Optional MISTRAL_OCR_MODEL env var overrides the model arg default
   - Construct `mistralai.Mistral(api_key=resolved_key, server_url=resolved_server_url)`; call `client.ocr.process(model=resolved_model, document={"type":"document_url","document_url":...})` (exact API shape per Mistral docs; SDK accepts file uploads)
   - Returns markdown string (concatenate per-page pages[].markdown if response is per-page)
   - Per-call telemetry: log page count + wall time + projected cost (cost per page = $0.001 for mistral-ocr family; document that Foundry MaaS may bill differently — log is informational, not authoritative)
   - Defensive error handling: catch mistralai-specific exceptions and re-wrap as DoclineError with path context
   - Validate server_url shape when set: must look like a URL (starts with https://). MistralCredentialError with clear message when malformed.
