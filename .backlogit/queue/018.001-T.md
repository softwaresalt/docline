Build a one-shot analysis script at `scripts/spike_h1_corpus_analysis.py` that parses every `.md` part under `.elt/output/` and reports, per source job: (a) count of parts with `section_title: null`; (b) count of parts whose body first heading is H2/H3 (no H1); (c) count of parts where `title` frontmatter could plausibly serve as H1 (length ≤ 100 chars, not "Untitled" / "Part N" placeholder); (d) count of parts where the first non-empty paragraph is a sensible candidate (length 10–120 chars, not a markdown image, not URL-only). Output: a structured JSON summary to stdout and a human-readable Markdown table. Use `with` context-managed file opens (plan-review advisory A1). Script must pass `ruff check .` and `ruff format --check .` since `scripts/` is in the ruff scope.
