title: Build OCR memory calibration harness (operator-run) to determine limits
updated_at: 2026-06-30T20:45:42.7167645-07:00
---

Build a calibration harness (e.g. scripts/study/ocr_memory_calibration.py) the operator runs to measure docling OCR peak memory and emit a PORTABLE, host-independent cost model — so docline can compute OCR page caps at runtime on any host relative to that host's available memory (operator design 2026-06-30).

What it measures: for representative pages spanning a range of bitmap sizes (megapixels), sweep docling render scale (ocr_scale, from 040.001-T) and pages-per-group, run docling OCR in the batched worker subprocess, and record peak RSS (psutil on the child process) + success/OOM. Because peak OCR memory for a given bitmap is largely hardware-independent (same computation, same working set), the harness fits a PORTABLE cost model, e.g. peak_mb ≈ base_mb + k_mb_per_mpx * (page_megapixels * scale^2) [times pages-in-group for accumulation]. Emit: (a) a TSV of {page_class, page_megapixels, scale, pages_per_group, peak_rss_mb, outcome}; (b) the fitted coefficients (base_mb, k_mb_per_mpx); (c) the measurement host's available memory (resource_probe) for context. Write a decision artifact (docs/decisions/) with the coefficients so the runtime algorithm (follow-up) can compute, for ANY host: max pages-per-group = safe_fraction * available_mb / per_page_cost, and the render scale that brings an oversized single page under budget.

Agent-buildable + unit-testable (036-S pattern): the harness LOGIC — arg parsing, sweep-orchestration seams, TSV emission, and especially the model-FIT + recommendation from a measurements table — is unit-tested by loading the script via importlib and feeding synthetic measurement rows (cf. tests/test_compare_merge_gap.py), asserting the fitted coefficients and derived caps. The real measurement RUN is operator-executed on a docling host; guard the docling/psutil measurement path behind availability checks so import + logic tests pass without docling. Depends on 040.001-T.

Do NOT hardcode a cap/schedule here; output the portable model. Applying it as a runtime host-relative algorithm is stash 699FB5DC.
