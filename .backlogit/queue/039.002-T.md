Extract the duplicated per-page OCR-gate loop shared by src/docline/process/pdf_triage.py::_range_needs_ocr (line ~262) and src/docline/process/pdf_batch.py::_chunk_needs_ocr (line ~220) into one shared helper. Both run the same logic: iterate pages, extract per-page text via pypdf, call fidelity_scorer.page_needs_ocr(text, page_metadata), and short-circuit with any(). They differ only in the input source (a pypdf reader + index range vs a chunk PDF path + reused heuristic text).

Implementation: add a shared helper (e.g. fidelity_scorer.any_page_needs_ocr(pairs: Iterable[tuple[str, object|None]]) -> bool, next to page_needs_ocr) that takes an iterable of (text, page_metadata) pairs and returns True if any page needs OCR. Have _range_needs_ocr build its (text, page) pairs (reusing heuristic_pages, falling back to page.extract_text() for empty slots — keep the existing conservative behavior) and delegate; have _chunk_needs_ocr open the chunk via pypdf and delegate. Preserve the conservative defaults (unreadable input keeps OCR on). No behavior change — existing tests/process/test_conditional_ocr.py must pass unchanged. Width: fidelity_scorer.py + pdf_triage.py + pdf_batch.py + their tests. Source: review F4 from 036-S.
