Refresh the `docline._tools.docling_worker` subprocess output contract to emit a JSON envelope (`{"pages":[...], "page_count":N, "text":"...", "schema_version":1}`) so the parent orchestrator can index by page instead of receiving one flat blob per range. Restores per-page fidelity in `pdf_triage` splice-back. Adds a multi-chunk batched worker mode that processes N chunks per subprocess invocation, sharing the docling model load to cut wall-clock on multi-chunk runs by an empirically-measured amount (cosmos PA3+PA4 baseline: 86 invocations × 24pp avg × 247min wall-clock). Probes the rt_detr layout-model batch size and commits a tuned constant if exposed. Validates per-page fidelity end-to-end on the cosmos sample. Consumes stash entries 5CFE4481 (per-page output limitation) and 51332802 (subprocess batch tuning). Architectural direction selected in deliberation 001-DL.
