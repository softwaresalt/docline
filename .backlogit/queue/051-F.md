title: Swagger 2.0 → OpenAPI 3.x spec pre-conversion
updated_at: 2026-07-05T21:46:20.5886776-07:00

Add a pre-conversion/normalization stage that upgrades non-3.x API specifications (Swagger 2.0 first) to OpenAPI 3.x BEFORE they reach the existing 3.x renderer (050-F), so a single ingestion path covers both spec generations. Immediately unblocks fabric-rest-api-specs, which the 2026-07-05 ingest run showed is 100% Swagger 2.0 (131 `swagger: "2.0"` specs, 0 OpenAPI 3.x) and therefore currently produces zero output. Supersedes the "Swagger 2.0 rendering" line item deferred in stash D9AC2CD6 by taking the pre-conversion approach (reuse the 3.x renderer) rather than writing a parallel 2.0 renderer. Preferred implementation: an in-pipeline Python converter (no heavy new dependency, Principle VI) that runs when content-sniff reports `SWAGGER_20`, transforms the model to OpenAPI 3.1, then feeds `read_openapi_spec`.

<!-- BEGIN:dod -->
- A Swagger 2.0 spec is detected (already: `openapi_file_kind == SWAGGER_20`), converted to OpenAPI 3.x, and rendered to per-operation/per-schema docs.
- Conversion mapping covers the core 2.0→3.x deltas: `swagger`→`openapi`; `definitions`→`components.schemas`; body/formData `parameters`→`requestBody`+`content`; `consumes`/`produces`→request/response `content` media types; response `schema`→response `content.*.schema`; `host`+`basePath`+`schemes`→`servers`; `securityDefinitions`→`components.securitySchemes`; `$ref #/definitions/X`→`#/components/schemas/X`.
- The process gate (`_is_openapi_staged`) and `read_openapi_spec` accept converted 2.0 specs (currently they require 3.x).
- Regression tests: a hand-written 2.0 spec converts + renders to valid BaseDocuments; fabric-rest-api-specs style body-param + definitions refs resolve.
- Quality gates pass (ruff, pyright, pytest, format).
- Re-run fabric-rest-api-specs end-to-end and confirm non-empty output.
<!-- END:dod -->

<!-- BEGIN:goals -->
- One ingestion path renders both OpenAPI 3.x and Swagger 2.0 specs to the same BaseDocument output contract.
- Unblock fabric-rest-api-specs (131 Swagger 2.0 specs) for the graphtor unified-DB test.
- Reuse the 050-F 3.x renderer via up-front conversion instead of a parallel 2.0 code path.
<!-- END:goals -->
