<!-- BEGIN:ImplementationNotes -->
Place normalization after `if not markdown or not markdown.strip(): return [""]` so empty/whitespace short-circuit is unaffected. Use chained `.replace()` calls — first `\r\n` (Windows) then `\r` (legacy Mac) to avoid double-conversion of Windows endings into `\n\n` (which would create spurious paragraph breaks).
