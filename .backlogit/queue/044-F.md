Derive and stamp a per-document canonical Microsoft Learn URL path (e.g. /fabric/admin/foo; forward-slash, host-less) during local-dir ingestion, so graphtor-docs can resolve cross-product links across separately-ingested repos. graphtor's planned Option B (cross-source edge resolution) keys on this globally-unique URL. docline is the only tool positioned to compute it — graphtor only sees post-docline standardized markdown, never the source repo's openpublishing/docfx publish config. Reframed from stash 4A650FFD ("multi-repo orchestration"), which the spike showed would NOT help on its own. Rationale + the graphtor-side feature spec: docs/decisions/2026-07-03-graphtor-cross-repo-link-resolution-spike.md.

<!-- BEGIN:dod -->
- Local-dir docs under a recognized publish config carry `canonical_url` under `docline:`; docs without a config are unchanged (no `canonical_url`).
- v1 derivation covers base path + url_path_prefix + file→URL normalization; monikers / redirect maps / documentId path-depot mappings are explicitly DEFERRED and documented.
- Unit + end-to-end tests cover derivation and stamping (incl. unmatched → None).
- All four quality gates pass: ruff check ., pyright src/, pytest, ruff format --check .
<!-- END:dod -->

<!-- BEGIN:goals -->
- Derive a per-doc canonical Learn URL path from the source repo's publish config (`.openpublishing.publish.config.json` / `docfx.json`) + the doc's repo-relative path.
- Stamp it under the docline frontmatter namespace so graphtor-docs Option B can key cross-source cross-product link resolution on it.
- Graceful no-op when no publish config is present (behavior unchanged).
<!-- END:goals -->
