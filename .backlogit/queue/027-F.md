SPIKE: Add Azure Document Intelligence (ADI) as third pdf_engine peer alongside docling and heuristic.

Strategic intent per deliberation docs/decisions/2026-06-10-next-pdf-pipeline-shipment-deliberation.md (ratified PR #66):

- Highest expected throughput improvement (cosmos PDF 25h -> 30min)
- Largest reduction in operational complexity (eliminates pdf_batch.py subprocess containment)
- Time-boxed spike scope — bounded risk; abandon if quality drops below 90% of docling
- ~$6.75 operational cost per cosmos run (Azure prebuilt-layout API at $1.50/1k pages)

Spike outcome decides whether ADI becomes the default high-fidelity engine OR stays available as opt-in. Both docling and heuristic remain as peer engines regardless.

Decomposed into 5 tasks (T1-T5) per the deliberation doc's recommended shipment shape (~11-16 hours total).

Acceptance criteria:
- AC1: T1+T2+T3 wire ADI as an opt-in pdf_engine without breaking existing docling/heuristic paths
- AC2: T4 empirical study on 5 cosmos sample ranges from .elt/output/cosmos-triage-022/study/results/ shows ADI structural density within 10% of docling, wall time <5% of docling, 0 per-page failures
- AC3: T5 closure doc with cost analysis + recommended auto policy (prefer ADI when AZURE_DOCUMENT_INTELLIGENCE_ENDPOINT set; fall back to docling, then heuristic)
- AC4: All four local quality gates pass (ruff/format/pyright/pytest); no test regressions
- AC5: Spike output decides production rollout — DOES NOT mandate it
