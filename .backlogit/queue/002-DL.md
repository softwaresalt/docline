# Conditional docling OCR via reused triage text-layer signal

## Problem Frame

<!-- BEGIN:problem-frame -->
docling's `do_ocr` is unset in `_read_pdf_docling_pages` (src/docline/readers/pdf.py:762), so it inherits docling's default of True. RapidOCR (onnxruntime PP-OCRv4) therefore runs on every page routed to docling, including native-text pages that need no OCR. Confirmed in the 6E6754D4 diagnostic: a single text page loaded det/cls/rec ONNX models and took minutes. This dominates per-page wall time (~247min cosmos CPU baseline) and amplifies the batched-mode memory pressure 033-S identified. Goal: skip OCR for pages that already carry an extractable text layer, without regressing genuine image-only/scanned PDFs (which is why a blanket do_ocr=False is wrong).
<!-- END:problem-frame -->

## Options

<!-- BEGIN:options -->
A) Range-level do_ocr threaded from the triage/batch layer into `_read_pdf_docling_pages` and the docling_worker dispatch. Compute per coalesced range from the per-page text-layer signal the triage pass ALREADY produces (pypdf `extract_text()` length / `signal_char_density`): range gets do_ocr=False when every page has a healthy text layer, True if any page is image-only (conservative). Lowest risk, reuses existing signals (no new extraction), fits the existing per-range coalescing. B) Per-page splitting by text-layer presence so text pages and image-only pages go to separate docling calls with their own do_ocr. Best fidelity on mixed PDFs but more complex (interacts with range coalescing + envelope splice-back). C) Use a docling-native option to OCR only bitmap regions / skip text pages. Least control; depends on docling API stability and still pays OCR-engine init.
<!-- END:options -->

## Chosen Direction

<!-- BEGIN:chosen-direction -->
Option A for the first slice. Add an explicit `do_ocr: bool` parameter to `_read_pdf_docling_pages` (set on PdfPipelineOptions) and thread a per-range `do_ocr` decision from `pdf_triage`/`pdf_batch` through the docling_worker manifest into the conversion. Derive the decision from the text-layer signal already computed in the triage Pass-1 heuristic/score pass — a range is do_ocr=False only when all its pages show a healthy extractable text layer; any image-only page forces do_ocr=True for that range (conservative, no fidelity regression). Reuses existing per-page signals so there is no extra extraction cost. Keep behavior backward-compatible: callers that do not pass a decision default to do_ocr=True (today's behavior), so the change is opt-in at the call sites that have triage signals. Defer Option B (per-page split) to a follow-up gated on a mixed-PDF corpus showing measurable fidelity loss.
<!-- END:chosen-direction -->
