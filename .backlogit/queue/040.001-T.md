title: Add ocr_scale knob through docling worker manifest + reader
updated_at: 2026-06-30T19:10:16.4301951-07:00
---

Add an optional per-invocation OCR render-scale knob to the docling worker path so a page can be rasterized at a lower effective DPI (less OCR peak memory).

Surface: src/docline/readers/pdf.py::_read_pdf_docling_pages currently sets PdfPipelineOptions(images_scale=2.0). Add an `ocr_scale: float | None = None` param; when provided, use it as the images_scale for the invocation (e.g. images_scale=ocr_scale) so the OCR engine receives a lower-resolution page bitmap. Keep the 2.0 default when None (no behavior change). src/docline/_tools/docling_worker.py: read an optional per-chunk `ocr_scale` float from the --batch manifest (default None; mirror how `do_ocr` is read) and forward it to _read_pdf_docling_pages; also accept it in single-chunk mode if practical (optional). This task is the plumbing + unit tests ONLY (batch_dispatch downscale-retry is task 2).

Tests: docling_worker manifest with an `ocr_scale` entry forwards the value to the reader (assert via a monkeypatched/fake _read_pdf_docling_pages capturing kwargs, consistent with existing worker tests in tests/tools/test_docling_worker.py); default omitted -> reader called without ocr_scale (or None). No docling install required (mock the reader). Width: src/docline/readers/pdf.py + src/docline/_tools/docling_worker.py + tests/tools/test_docling_worker.py.

<!-- BEGIN:acceptance-criteria -->
- _read_pdf_docling_pages accepts ocr_scale; when set, images_scale uses it; default (None) keeps images_scale=2.0.
- docling_worker reads optional per-chunk ocr_scale from the manifest and forwards it to the reader; absent -> not forwarded / None.
- Unit tests in tests/tools/test_docling_worker.py assert forwarding + default, with the docling reader mocked (no docling install needed).
- Gates green: ruff, pyright src/ (0 errors), pytest, ruff format --check.
