<!-- BEGIN:acceptance-criteria -->
- New `QASampling` frozen dataclass groups `sample_rate: float`, `random_seed: int | None`, `max_sampled_pages: int = 50` — passed to `process_pdf_triaged` as a single optional kwarg (plan-review P2 finding: avoid kwargs explosion).
- `qa_sampling=None` (default): runner invoked only for flagged ranges.
- `qa_sampling.sample_rate=1.0`: runner invoked for every unflagged page (subject to `max_sampled_pages` cap).
- Disagreement counter increments when sampled-docling output differs from heuristic for the same page.
- Random seed = `None` means system-clock seed (`random.Random()`); explicit integer = deterministic. Resolved seed always recorded in `TriageResult.metadata['qa_random_seed_used']` (plan-review P3 finding).
- CLI exposure: `--sample-rate FLOAT` and `--qa-random-seed INT`; `--sample-rate 0` disables (default).
- RED tests written first and fail; GREEN after implementation.
- Test scenarios: (1) `sample_rate=0.0` invokes runner only for flagged ranges; (2) `sample_rate=1.0` invokes runner for every unflagged page (subject to cap); (3) disagreement counter increments when outputs differ; (4) random seed makes sample selection deterministic.
<!-- END:acceptance-criteria -->

<!-- BEGIN:description -->
Add an optional QA tripwire mode to `process_pdf_triaged` that randomly re-runs a configurable fraction of unflagged ('clean') pages through docling and diffs the output against heuristic. Disagreement count is recorded in `TriageResult.metadata['qa_disagreements']`.

Purpose: catch false-negative under-flagging — pages where the scorer said 'no docling needed' but docling output actually differs materially. Disagreement count becomes a regression-detection metric for production runs.

Files:
- `src/docline/process/pdf_triage.py` (modified — add `QASampling` dataclass kwarg)
- `src/docline/cli.py` (modified — add `--sample-rate FLOAT` and `--qa-random-seed INT` flags)
- `tests/process/test_pdf_triage_tripwire.py` (new)

Depends on: U3 (orchestrator).
<!-- END:description -->

<!-- BEGIN:implementation-notes -->
Use `random.Random(seed)` (a local Random instance), NOT module-level `random.seed()` — module-level seeding has cross-test contamination risks.

Diff function: compare normalized markdown (strip trailing whitespace per line, collapse blank-line runs) so trivial whitespace variation does not inflate disagreement count.

The `max_sampled_pages` cap defaults to 50 to bound runtime on long docs (per plan hardening); operator can raise for QA-focused runs.
