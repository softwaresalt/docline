Add a first-class OpenAPI/Swagger ingestion path that renders structured Markdown from an authoritative spec (not the lossy PDF pipeline). v1 scope, green-lit by operator 2026-07-05: OpenAPI 3.x only, single-spec (file or one-service directory), per-operation granularity, local #/components/* $ref only. Design + T1-T6 decomposition from spike docs/decisions/2026-07-04-openapi-swagger-source-type-spike.md (verdict: PROCEED). Reuses BaseDocument/BaseFrontmatter output contract and existing PyYAML/pydantic deps (no new dependency). Sibling to spike feature 049-F.

<!-- BEGIN:dod -->
- OpenAPI 3.x specs are detected by content-sniff and routed to a dedicated `readers/openapi.py`; plain config `.json` is not misclassified.
- One Markdown document is emitted per operation and per named component schema; emitted docs pass `BaseDocument` schema validation.
- Local `#/components/*` `$ref` resolution works with a cycle guard; external/split-file refs are explicitly deferred.
- CLI (`docline process`) and MCP surfaces produce identical output for one spec (dual-interface parity); `--manifest` advertises the path; README documents it.
- All six tasks (T1-T6) pass `ruff check .`, `pyright src/`, `pytest`, and `ruff format --check .`.
- Deferred-beyond-v1 items are recorded as a follow-up stash entry on completion.
<!-- END:dod -->

<!-- BEGIN:goals -->
- Render deterministic Markdown per REST API operation from an authoritative OpenAPI 3.x spec, replacing the lossy PDF path for REST reference content.
- Preserve referentiality: emit each `$ref` as an `operation -> schema` Markdown link so graphtor harvests typed edges.
- Fit the existing `BaseDocument`/`BaseFrontmatter` output contract with no new dependency (reuse PyYAML + pydantic).
- Keep the existing FILE path untouched; make the OpenAPI pipeline explicit and independently testable.
<!-- END:goals -->
