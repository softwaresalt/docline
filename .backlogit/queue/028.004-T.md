Build scripts/study/validate_pre_triage.py — empirical validation that the new pre-triage classifications match docling's wins from the 2026-06-08 extraction study.

Inputs:
- The cosmos study fixture pages from .elt/output/cosmos-triage-022/study/results/ (15 sample ranges from the existing baseline)
- The docling-winners ground truth from docs/decisions/2026-06-08-extraction-strategy-study.md findings.json

Process per page:
1. Load the source PDF page via pypdf
2. Run pre_triage_score
3. Compare classification to ground truth:
   - Ground truth says "docling wins" + classification == "route_to_docling" → agreement++
   - Ground truth says "heuristic wins" + classification == "route_to_heuristic" → agreement++
   - "uncertain" classification counts as neutral (split 50/50)

Output: validation-report.json with per-page rows + aggregate agreement percentage. Exit non-zero if aggregate < 85%.

Then write docs/closure/030-S-invert-triage-scoring.md documenting:
- What was built (architecture, new flag, opt-in semantics)
- Validation results (agreement percentage, per-signal correlation if computed)
- Rollout recommendation: ADOPT (make --triage-pre-score default-on), KEEP-AS-OPT-IN (operator opt-in only), or ABANDON (insufficient agreement)
- Follow-up stash candidates (e.g. promote to default in --pdf-mode auto, signal weight calibration, etc.)
