In src/docline/process/pdf_triage.py splice-back post-pass (~lines 664-691), a single-entry envelope for an N-page docling range is the EXPECTED whole-range render (docling's coherent export_to_markdown over the range; the collapse branch already assigns the envelope 'text' = join of pages). Today this path logs an "envelope length mismatch" WARNING (thousands of noise lines on real runs) and attributes engine="docling-collapsed".

REFRAME (2026-06-28 merge-gap verdict; output_contract.py:264 joins pages and assemble.py:141-144 runs whole-body heading/AST validation): a per-page split would FRAGMENT structures and worsen AST/heading/lint, so the whole-range blob is CORRECT. Do NOT implement a per-page split.

Fix: when an N(>1)-page range's envelope has a single coherent entry (the expected whole-range render), assign the blob to the range WITHOUT the mismatch WARNING and relabel attribution to "docling-range" (not pejorative "docling-collapsed"). Keep a genuine WARNING only for truly unexpected envelope shapes (e.g. multi-entry but wrong count). Update scripts/pa3_triage_cosmos.py::_docling_attribution to report ranges (not per-page "docling-collapsed" placeholders) so the internal routing metric is accurate. No change to assembled markdown / AST / headings / referentiality. Files: src/docline/process/pdf_triage.py, scripts/pa3_triage_cosmos.py (+ tests). Refs: docs/decisions/2026-06-28-merge-gap-tuning-verdict.md.
